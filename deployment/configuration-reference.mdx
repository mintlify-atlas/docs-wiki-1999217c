All Fineract runtime configuration is declared in `fineract-provider/src/main/resources/application.properties`. Every property supports an environment variable override using the Spring `${ENV_VAR:default}` syntax. The sections below document every property grouped by functional area.

<Tip>
Set any property at runtime by either (a) exporting the named environment variable, (b) passing `-DENV_VAR=value` as a JVM argument, or (c) supplying an `application.properties` override file at `--spring.config.additional-location`.
</Tip>

| Property | Env Var | Default | Description |
|---|---|---|---|
| `fineract.node-id` | `FINERACT_NODE_ID` | `1` | Numeric node identifier. Must be unique across all instances in a cluster. Used for partitioned job worker assignment. |

## Security

### Basic Authentication

| Property | Env Var | Default | Description |
|---|---|---|---|
| `fineract.security.basicauth.enabled` | `FINERACT_SECURITY_BASICAUTH_ENABLED` | `true` | Enable HTTP Basic authentication. |
| `fineract.security.oauth2.enabled` | `FINERACT_SECURITY_OAUTH_ENABLED` | `false` | Enable OAuth2 authentication (disables Basic). |
| `fineract.security.2fa.enabled` | `FINERACT_SECURITY_2FA_ENABLED` | `false` | Enable two-factor authentication. |
| `fineract.security.hsts.enabled` | `FINERACT_SECURITY_HSTS_ENABLED` | `false` | Add `Strict-Transport-Security` header to all responses. |

### CORS

| Property | Env Var | Default | Description |
|---|---|---|---|
| `fineract.security.cors.enabled` | `FINERACT_SECURITY_CORS_ENABLED` | `true` | Enable CORS filter. |
| `fineract.security.cors.allowed-origin-patterns` | `FINERACT_SECURITY_CORS_ALLOWED_ORIGIN_PATTERNS` | `*` | Comma-separated origin patterns (supports wildcards). |

| `fineract.security.cors.allowed-headers` | `FINERACT_SECURITY_CORS_ALLOWED_HEADERS` | `*` | Allowed request headers. |

### OAuth2 Client Registration Example

```properties
# Frontend client registration (example values)
fineract.security.oauth2.client.registrations.frontend-client.client-id=${FINERACT_SECURITY_OAUTH2_CLIENTS_FRONTEND_ID:frontend-client}
fineract.security.oauth2.client.registrations.frontend-client.scopes=${FINERACT_SECURITY_OAUTH2_CLIENTS_FRONTEND_SCOPES:read,write}
fineract.security.oauth2.client.registrations.frontend-client.authorization-grant-types=${FINERACT_SECURITY_OAUTH2_CLIENTS_FRONTEND_GRANTS:authorization_code,refresh_token}
fineract.security.oauth2.client.registrations.frontend-client.redirect-uris=${FINERACT_SECURITY_OAUTH2_CLIENTS_FRONTEND_REDIRECT:http://localhost:3000/callback}
fineract.security.oauth2.client.registrations.frontend-client.require-authorization-consent=${FINERACT_SECURITY_OAUTH2_CLIENTS_FRONTEND_CONSENT:false}
```

## Mode Flags

Mode flags enable horizontal scaling by designating each node as a reader, writer, batch manager, or batch worker.

| Property | Env Var | Default | Description |
|---|---|---|---|
| `fineract.mode.read-enabled` | `FINERACT_MODE_READ_ENABLED` | `true` | Accept read-only API requests. |
| `fineract.mode.write-enabled` | `FINERACT_MODE_WRITE_ENABLED` | `true` | Accept write (mutation) API requests. |
| `fineract.mode.batch-worker-enabled` | `FINERACT_MODE_BATCH_WORKER_ENABLED` | `true` | Process incoming batch job chunks. |
| `fineract.mode.batch-manager-enabled` | `FINERACT_MODE_BATCH_MANAGER_ENABLED` | `true` | Schedule and partition batch jobs. |

<Note>
In a manager/worker topology, the manager runs with `batch-worker-enabled=false` and the workers run with `batch-manager-enabled=false` and read/write disabled.
</Note>
