Apache Fineract is an open-source core banking platform that provides a flexible, extensible foundation for delivering financial services to underserved and unbanked populations. The project is incubated under the Apache Software Foundation and is released under the Apache License 2.0. This document is an engineering orientation: what the system does, how it is structured as a codebase, and what runtime dependencies it brings.

## What Fineract Provides

Fineract exposes a REST API (backed by Jersey/JAX-RS) covering the following functional domains:

<CardGroup cols={2}>
  <Card title="Loan Products" icon="hand-holding-dollar">
    Full loan lifecycle — origination, approval, disbursement, repayment scheduling, interest calculation, and Close-of-Business (COB) batch processing. Progressive loan and working-capital loan modules are optional.
  </Card>
  <Card title="Savings & Deposits" icon="piggy-bank">
    Savings accounts, fixed and recurring deposits with interest posting, charges, and product configuration through the `fineract-savings` module.
  </Card>
  <Card title="Accounting" icon="book">
    Double-entry accounting with chart-of-accounts management, journal entries, financial activity mappings, and periodic accruals through `fineract-accounting`.
  </Card>
  <Card title="Multi-Tenancy" icon="building">
    Full per-tenant database isolation. Each tenant has its own schema (or database) with independent Liquibase-managed migrations on startup.
  </Card>
  <Card title="Security" icon="shield">
    Pluggable authentication: HTTP Basic, OAuth2, and optional two-factor auth through `fineract-security`. Role-based permissions per resource/action.
  </Card>
  <Card title="Jobs & COB" icon="clock">
    Spring Batch–driven scheduled jobs, including Close-of-Business processing. Supports local and distributed (partitioned) job execution via ActiveMQ or Kafka.
  </Card>
</CardGroup>

## Technology Stack

| Layer | Technology |
|---|---|
| Runtime | Java 21 (Azul Zulu recommended), embedded Tomcat via Spring Boot |
| Web Framework | Spring Boot 3.x with Jersey (JAX-RS) for REST endpoints |
| Persistence | JPA/Hibernate, Spring Data, HikariCP connection pooling |
| Database | PostgreSQL ≥ 18 (primary); MariaDB/MySQL deprecated |
| Migrations | Liquibase — applied per-tenant on startup |
| Build | Gradle multi-project build (`settings.gradle`) |
| Messaging | Spring Events (default), Apache ActiveMQ, Apache Kafka (optional) |
| API Docs | OpenAPI 3 / Swagger UI at `/fineract-provider/swagger-ui/index.html` |

## Running Fineract Locally

The fastest path to a running instance requires Java 21 and PostgreSQL:

```bash
git clone https://github.com/apache/fineract.git
cd fineract

# Create the two required databases
./gradlew createPGDB -PdbName=fineract_tenants
./gradlew createPGDB -PdbName=fineract_default

# Start the embedded-Tomcat Spring Boot server
./gradlew devRun
```

After startup the API is available at `https://localhost:8443/fineract-provider/api/v1/`. Every request must carry the `Fineract-Platform-TenantId` header (e.g. `default`) and an `Authorization` header.

<Tip>
Override any default from `fineract-provider/src/main/resources/application.properties` via environment variables or Java system properties. Environment variables use `FINERACT_` prefix with underscores replacing dots (e.g. `FINERACT_HIKARI_JDBC_URL`).
</Tip>

<Note>
A self-contained bootJar is built with `./gradlew clean bootJar`. The output lands in `fineract-provider/build/libs/`. The official Docker image at `apache/fineract` on Docker Hub is built from the same JAR.
</Note>

|---|---|
